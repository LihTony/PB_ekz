📚 Распределенные вычисления

Подготовка к экзамену

1. Архитектурный подход REST API (CRUD), RESTful веб-сервисы
REST (Representational State Transfer) — архитектурный стиль для создания веб-сервисов, основанный на HTTP.
  • CRUD операции: Create (POST), Read (GET), Update (PUT/PATCH), Delete (DELETE)
  • Принципы REST: stateless (без состояния), клиент-сервер, кэширование, единообразие интерфейса
  • RESTful сервисы используют HTTP методы, работают с ресурсами через URI, возвращают данные в JSON/XML
  • Пример: GET /users/123 — получить пользователя, POST /users — создать нового
2. Понятие протокол. Виды протоколов
Протокол — набор правил для обмена данными между устройствами в сети.
  • Прикладные: HTTP, FTP, SMTP, DNS
  • Транспортные: TCP, UDP
  • Сетевые: IP, ICMP, ARP
  • Канальные: Ethernet, Wi-Fi
3. Протоколы HTTP/HTTPS
HTTP (HyperText Transfer Protocol) — протокол передачи гипертекста, работает на порту 80.
  • Методы: GET, POST, PUT, DELETE, HEAD, OPTIONS
  • Коды ответов: 2xx (успех), 3xx (перенаправление), 4xx (ошибка клиента), 5xx (ошибка сервера)
  • HTTPS — защищенная версия HTTP с шифрованием SSL/TLS, порт 443
  • Отличие: HTTPS шифрует данные, защищает от перехвата
4. SOAP, структура протокола
SOAP (Simple Object Access Protocol) — протокол обмена структурированными сообщениями на основе XML.
  • Структура: Envelope (конверт) → Header (заголовок) → Body (тело) → Fault (ошибки)
  • Envelope: корневой элемент, определяет начало и конец сообщения
  • Header: метаданные (аутентификация, маршрутизация)
  • Body: основные данные запроса/ответа
  • Fault: информация об ошибках
  • Отличие от REST: более строгий, использует только XML, поддерживает WS-Security
5. Механизм NAT
NAT (Network Address Translation) — преобразование IP-адресов для связи локальной сети с интернетом.
  • Назначение: экономия публичных IP-адресов, скрытие внутренней структуры сети
  • Принцип: заменяет частные IP (192.168.x.x) на публичный IP при выходе в интернет
  • Типы: Static NAT (1:1), Dynamic NAT (пул адресов), PAT/NAPT (много устройств → 1 IP через порты)
6. Служба DNS
DNS (Domain Name System) — служба преобразования доменных имен в IP-адреса.
  • Назначение: связывает понятные имена (google.com) с IP-адресами (142.250.185.46)
  • Иерархия: корневые серверы → серверы доменов верхнего уровня (.com, .ru) → авторитетные серверы
  • Типы записей: A (IPv4), AAAA (IPv6), MX (почта), CNAME (псевдоним), NS (сервер имен)
  • Порт: 53 (UDP/TCP)
7. Служба DHCP
DHCP (Dynamic Host Configuration Protocol) — автоматическое назначение IP-адресов устройствам в сети.
  • Назначение: упрощает настройку сети, избегает конфликтов IP-адресов
  • Процесс: Discover (поиск) → Offer (предложение) → Request (запрос) → Acknowledge (подтверждение)
  • Выдает: IP-адрес, маску подсети, шлюз по умолчанию, DNS-серверы
  • Порты: 67 (сервер), 68 (клиент)
8. Протоколы TCP/UDP
TCP (Transmission Control Protocol) — надежный протокол с установкой соединения.
  • TCP: гарантирует доставку, порядок пакетов, контроль ошибок, медленнее (HTTP, FTP, SMTP)
  • Установка соединения: 3-way handshake (SYN → SYN-ACK → ACK)
UDP (User Datagram Protocol) — быстрый протокол без установки соединения.
  • UDP: не гарантирует доставку, быстрее, меньше накладных расходов (DNS, видео, игры)
  • Отличия: TCP — надежность, UDP — скорость
9. Безопасность в сетях
  • Конфиденциальность: защита данных от несанкционированного доступа (шифрование)
  • Аутентификация: проверка подлинности пользователя (пароли, сертификаты)
  • Авторизация: предоставление прав доступа к ресурсам
  • Целостность: защита от изменения данных (хеш-функции, цифровые подписи)
  • Криптография: наука о шифровании данных
  • Криптоанализ: взлом шифров
  • Криптология: криптография + криптоанализ
  • Шифр: алгоритм преобразования данных
  • Код: замена слов/фраз на символы
  • Ключ шифра: секретная информация для шифрования/дешифрования
  • IPsec: защита IP-пакетов на сетевом уровне
  • SSL/TLS: шифрование на транспортном уровне (HTTPS)
  • HTTPS: HTTP + SSL/TLS
  • DNSsec: защита DNS от подделки записей
10. Модели взаимодействия сетевых приложений (OSI/ISO, TCP/IP)
OSI (7 уровней):
  • 7. Прикладной (HTTP, FTP)
  • 6. Представления (шифрование, сжатие)
  • 5. Сеансовый (управление сеансами)
  • 4. Транспортный (TCP, UDP)
  • 3. Сетевой (IP, маршрутизация)
  • 2. Канальный (Ethernet, MAC-адреса)
  • 1. Физический (кабели, сигналы)
TCP/IP (4 уровня):
  • 4. Прикладной (HTTP, DNS, FTP)
  • 3. Транспортный (TCP, UDP)
  • 2. Интернет (IP)
  • 1. Канальный (Ethernet)
11. Архитектура распределенного приложения (клиент/сервер)
Клиент-сервер — модель взаимодействия, где клиент запрашивает услуги, а сервер их предоставляет.
  • Клиент: инициирует запросы, имеет пользовательский интерфейс
  • Сервер: обрабатывает запросы, управляет ресурсами, хранит данные
  • Типы: двухуровневая (клиент-сервер), трехуровневая (клиент-сервер приложений-БД), многоуровневая
  • Преимущества: централизованное управление, масштабируемость, безопасность
  • Примеры: веб-браузер и веб-сервер, почтовый клиент и почтовый сервер
12. Понятие сокета. Основные параметры сокета
Сокет — программный интерфейс для обмена данными между процессами через сеть.
  • Определение: конечная точка двунаправленного канала связи
  • Параметры:
  • - IP-адрес (идентификация узла)
  • - Порт (идентификация приложения, 0-65535)
  • - Протокол (TCP или UDP)
  • - Тип сокета (SOCK_STREAM для TCP, SOCK_DGRAM для UDP)
  • Операции: socket() → bind() → listen() → accept() → send()/recv() → close()
13. Виртуализация в РИС
Виртуализация — создание виртуальных версий ресурсов (серверов, сетей, хранилищ).
  • Типы:
  • - Аппаратная (гипервизоры: VMware, Hyper-V, KVM)
  • - Контейнеризация (Docker, Kubernetes)
  • - Виртуализация приложений
  • Преимущества: эффективное использование ресурсов, изоляция, быстрое развертывание, миграция
  • В РИС: позволяет динамически распределять ресурсы между узлами
14. Синхронизация в РИС
Синхронизация — координация выполнения процессов для обеспечения корректной работы.
  • Проблемы: гонки данных, взаимоблокировки, несогласованность
  • Механизмы:
  • - Мьютексы (взаимное исключение)
  • - Семафоры (счетчики доступа)
  • - Мониторы (высокоуровневая синхронизация)
  • - Барьеры (ожидание всех процессов)
  • Синхронизация времени: NTP (Network Time Protocol)
  • Алгоритмы: Lamport timestamps, векторные часы, алгоритм Беркли
15. Виды и типы РИС
РИС (Распределенные Информационные Системы) — системы с компонентами на разных узлах сети.
  • По архитектуре:
  • - Клиент-сервер
  • - Peer-to-peer (P2P)
  • - Многоуровневые
  • По назначению:
  • - Распределенные БД
  • - Распределенные файловые системы (HDFS, GFS)
  • - Распределенные вычислительные системы (кластеры, грид)
  • По связности: слабосвязанные, сильносвязанные
16. Преимущества распределенных вычислений
  • Масштабируемость: легко добавлять новые узлы
  • Производительность: параллельная обработка задач
  • Надежность: отказ одного узла не останавливает систему
  • Доступность: данные доступны из разных мест
  • Экономичность: использование обычных компьютеров вместо суперкомпьютеров
  • Географическое распределение: размещение ресурсов ближе к пользователям
  • Отказоустойчивость: репликация данных
17. Использование распределенных вычислений
  • Научные расчеты: моделирование климата, анализ генома, физические эксперименты
  • Big Data: обработка больших объемов данных (Hadoop, Spark)
  • Веб-сервисы: Google, Facebook, Amazon
  • Финансы: анализ рынков, обработка транзакций
  • Машинное обучение: обучение нейронных сетей
  • Рендеринг: обработка видео и графики
  • Блокчейн: распределенные реестры (Bitcoin, Ethereum)
  • CDN: доставка контента (Cloudflare, Akamai)
18. Параллельные/сетевые вычисления
Параллельные вычисления — одновременное выполнение задач на нескольких процессорах.
  • Типы параллелизма:
  • - Параллелизм данных (одна операция на разных данных)
  • - Параллелизм задач (разные операции одновременно)
  • Архитектуры: многоядерные процессоры, GPU, кластеры
Сетевые вычисления — вычисления на узлах, связанных через сеть.
  • Отличие: параллельные — на одной машине/кластере, сетевые — через интернет
  • Примеры: SETI@home, Folding@home (добровольные вычисления)
19. Облачные РИС
Облачные вычисления — предоставление вычислительных ресурсов через интернет.
  • Модели обслуживания:
  • - IaaS (Infrastructure): виртуальные машины (AWS EC2, Azure VM)
  • - PaaS (Platform): платформы разработки (Google App Engine, Heroku)
  • - SaaS (Software): готовые приложения (Gmail, Office 365)
  • Модели развертывания: публичные, частные, гибридные облака
  • Преимущества: оплата по факту, эластичность, доступность, отсутствие обслуживания
  • Провайдеры: AWS, Microsoft Azure, Google Cloud, Yandex Cloud
20. Виды системных архитектур РИС
  • Клиент-сервер: централизованный сервер обслуживает клиентов
  • Многоуровневая (N-tier): разделение на уровни (презентация, бизнес-логика, данные)
  • Peer-to-peer (P2P): равноправные узлы (BitTorrent, Skype)
  • Микросервисы: приложение из независимых сервисов
  • Service-Oriented Architecture (SOA): взаимодействие через сервисы
  • Event-driven: реакция на события (Kafka, RabbitMQ)
  • Grid computing: объединение гетерогенных ресурсов
  • Cluster computing: группа однородных компьютеров
  • Serverless: выполнение функций без управления серверами (AWS Lambda)
